# Team Installation Guide

## 📋 Prerequisites

Each team member needs to install these tools first:

### Required Software

1. **PHP** (version 8.1 or higher)
   - Windows: Download from https://windows.php.net/download/
   - Or use XAMPP/WAMP which includes PHP
   - Verify: `php --version`

2. **Composer** (PHP package manager)
   - Download from https://getcomposer.org/download/
   - Verify: `composer --version`

3. **Node.js** (version 16 or higher)
   - Download from https://nodejs.org/
   - Includes npm automatically
   - Verify: `node --version` and `npm --version`

4. **MySQL** (or MariaDB)
   - Windows: Use XAMPP or MySQL Installer
   - Download XAMPP: https://www.apachefriends.org/
   - Or MySQL: https://dev.mysql.com/downloads/installer/

5. **Git** (for cloning repository)
   - Download from https://git-scm.com/
   - Verify: `git --version`

## 🚀 Setup Steps for Team Members

### Step 1: Clone the Repository

```bash
# Clone the project
git clone <your-repository-url>
cd <project-folder>
```

### Step 2: Backend Setup (Laravel)

```bash
# Navigate to backend folder
cd backend

# Install PHP dependencies
composer install

# Copy environment file
copy .env.example .env
# On Mac/Linux: cp .env.example .env

# Generate application key
php artisan key:generate

# Create database
# Option A: Using MySQL command line
mysql -u root -p
CREATE DATABASE event_management;
exit;

# Option B: Using phpMyAdmin (if using XAMPP)
# Go to http://localhost/phpmyadmin
# Create new database named "event_management"

# Run database migrations
php artisan migrate

# Create storage link
php artisan storage:link

# Start the backend server
php artisan serve
# Backend will run at: http://localhost:8000
```

### Step 3: Frontend Setup (React)

```bash
# Open a NEW terminal/command prompt
# Navigate to frontend folder
cd frontend

# Install Node.js dependencies
npm install

# Copy environment file
copy .env.example .env
# On Mac/Linux: cp .env.example .env

# Start the frontend development server
npm run dev
# Frontend will run at: http://localhost:5173
```

## 📦 What Gets Installed

### Backend (composer install)
This installs all PHP packages defined in `composer.json`:
- Laravel framework
- Laravel Sanctum (authentication)
- Database drivers
- Other PHP dependencies

**Size**: ~50-100 MB
**Time**: 2-5 minutes depending on internet speed

### Frontend (npm install)
This installs all JavaScript packages defined in `package.json`:
- React
- React Router
- Axios (API calls)
- Tailwind CSS
- QR Code library
- Supabase client
- Other JavaScript dependencies

**Size**: ~200-400 MB (node_modules folder)
**Time**: 3-10 minutes depending on internet speed

## ⚠️ Common Issues and Solutions

### Issue 1: "composer: command not found"
**Solution**: Composer not installed or not in PATH
```bash
# Download and install Composer from:
https://getcomposer.org/download/
```

### Issue 2: "npm: command not found"
**Solution**: Node.js not installed
```bash
# Download and install Node.js from:
https://nodejs.org/
```

### Issue 3: "php artisan: command not found"
**Solution**: PHP not installed or not in PATH
```bash
# Check if PHP is installed:
php --version

# If not, install PHP or add to PATH
```

### Issue 4: Database connection error
**Solution**: MySQL not running or wrong credentials
```bash
# Start MySQL (XAMPP):
# Open XAMPP Control Panel → Start MySQL

# Or check MySQL service is running
# Windows: services.msc → MySQL → Start

# Verify credentials in backend/.env match your MySQL setup
```

### Issue 5: "Port 8000 already in use"
**Solution**: Another process using port 8000
```bash
# Use different port:
php artisan serve --port=8001

# Update frontend .env:
VITE_API_URL=http://localhost:8001/api
```

### Issue 6: "Port 5173 already in use"
**Solution**: Another Vite server running
```bash
# Kill the process or use different port:
npm run dev -- --port 5174

# Access at: http://localhost:5174
```

### Issue 7: Composer install fails
**Solution**: PHP extensions missing
```bash
# Enable required extensions in php.ini:
extension=fileinfo
extension=pdo_mysql
extension=mbstring
extension=openssl

# Restart after editing php.ini
```

### Issue 8: npm install fails
**Solution**: Clear npm cache
```bash
npm cache clean --force
npm install
```

## 🔄 Daily Workflow

### Starting Work
```bash
# Terminal 1 - Backend
cd backend
php artisan serve

# Terminal 2 - Frontend
cd frontend
npm run dev
```

### Pulling Latest Changes
```bash
# Pull from repository
git pull

# Update backend dependencies (if composer.json changed)
cd backend
composer install
php artisan migrate  # If new migrations

# Update frontend dependencies (if package.json changed)
cd frontend
npm install
```

### Stopping Servers
```bash
# In each terminal, press:
Ctrl + C
```

## 🔐 Security Notes

### What to Share with Team
✅ Repository URL
✅ Supabase URL and anon key (already in .env.example)
✅ Database name (event_management)
✅ Setup instructions

### What NOT to Share Publicly
❌ Supabase service role key (keep in .env, not in git)
❌ APP_KEY (each member generates their own)
❌ Database passwords
❌ Any production credentials

## 📝 Summary

1. Install prerequisites (PHP, Composer, Node.js, MySQL)
2. Clone repository
3. Run `composer install` in backend/
4. Run `npm install` in frontend/
5. Configure .env files
6. Create database and run migrations

### Daily Workflow
1. Start MySQL (if not auto-start)
2. Run `php artisan serve` in backend/
3. Run `npm run dev` in frontend/
4. Start coding!

1. Run `composer install` if composer.json changed
2. Run `npm install` if package.json changed
3. Run `php artisan migrate` if new migrations
4. Restart servers
